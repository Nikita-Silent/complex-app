Это бэкэнд приложение для работы с api других приложений
---
Пример приложения: 
Я зашел в приложение - авторизовался через Telegram или authentik
Если пользователя нет, то происходит редирект на форму регистрации
Регистрация доступна через Telegram или authentik, после регистрации идет редирект на авторизацию
После успешной авторизации
У меня открывается домашняя панель с несколькими вкладками - отдельными интерфейсами (каждая вкладка ведет на отдельный микросервис)
Открывая вкладку tasks - Я могу создать задачу и назначить ее пользователю, который зарегистрирован в системе
Так же на вкладке tasks, я могу посмотреть на задачи в нескольких видах - kanban, диаграмма Ганта и список.
Открывая вкладку новости - мне открывается список новостей. Новости получаются из бд. В бд новости попадают по вебхуку (на него приходят новости).
Открывая вкладку личная информация - у меня открывается поле с возможностью редактирования информации о себе и так же несколько полей: Имя, фамилия, номер телефона, есть кнопка запросить штрихкод сотрудника, есть кнопка привязать Telegram (если был вход через authentik), есть кнопка привязать authentik (если был вход через Telegram) обе кнопки показывают статус (привязан/не привязан)
Открывая вкладку Сканер - можно выбрать endpoint куда будет идти запрос (все запросы идут на один fqdn, но различные uri) и нажать кнопку включения сканера. После сканирования отсканированные данные отправляются на endpoint  и выводится response.
